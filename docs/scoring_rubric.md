# 评分 Rubric — 面试评估标准

## 一、单题评分标准（1-10 分）

Evaluator Agent 和 Answer Assessor 共用此标准。

| 分数段 | 等级 | 判定标准 |
|--------|------|---------|
| **9-10** | 优秀 | 回答全面且深入，覆盖所有参考要点，有超出预期的洞见或实际经验分享 |
| **7-8** | 良好 | 回答准确完整，覆盖大部分参考要点（≥80%），表述清晰有条理 |
| **5-6** | 及格 | 回答基本正确，覆盖部分参考要点（40-80%），存在一些遗漏或表述不够精确 |
| **3-4** | 不足 | 回答部分正确但有明显错误或重大遗漏，仅覆盖少量要点（<40%） |
| **1-2** | 较差 | 回答基本错误或完全无关，未触及任何参考要点 |

## 二、追问触发规则

Answer Assessor 根据以下规则决定是否追问：

```
if score < 4 and follow_up_count == 0:
    → 追问（引导候选人往正确方向思考）
    
elif 4 <= score <= 6 and missed_points 非空 and follow_up_count == 0:
    → 追问一次（针对遗漏的关键点提问）
    
elif score >= 7:
    → 不追问，直接进入下一题
    
elif follow_up_count >= max_follow_ups:
    → 不再追问，强制进入下一题（防止死循环）
```

## 三、综合评估标准

Evaluator Agent 使用以下维度生成最终报告：

### 各技能维度评分
- 根据 SkillMatrix 中的技能列表逐一评分
- 每个维度综合该技能相关所有问题的回答质量
- 必须引用具体的对话内容作为证据

### 综合评分计算
```
overall_score = Σ (skill_score_i × weight_i) / Σ weight_i
```

### 等级划分
| 等级 | 分数范围 | 含义 |
|------|---------|------|
| **A** | ≥ 8.0 | 强烈推荐，具备岗位所需的核心能力 |
| **B** | 6.5 - 7.9 | 推荐，大部分能力达标，少数领域需加强 |
| **C** | 5.0 - 6.4 | 有条件推荐，基础尚可但多个领域需要提升 |
| **D** | < 5.0 | 暂不推荐，建议候选人进一步学习后重试 |

### 报告内容要求
1. **技能评分**：每个技能 1-10 分，必须附带对话证据
2. **优势**（3 条）：具体的亮点，引用回答中的原话
3. **改进建议**（3 条）：可操作的建议，指出具体的知识缺口
4. **总结**：2-3 句话的整体评价

## 四、评分一致性保障

### Prompt 层面
- 评分 Rubric 写入 System Prompt，确保每次评估使用相同标准
- 使用 Structured Output（JSON Schema）强制输出格式
- Few-shot 示例提供评分参考基准

### 工程层面
- `scripts/evaluate.py` 对同一回答跑多次，检查评分方差
- 可接受方差范围：单题评分 ±1 分，综合评分 ±0.5 分
- 超出方差范围需检查 Prompt 和 temperature 设置
